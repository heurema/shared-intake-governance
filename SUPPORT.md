# Support

## Where to ask for help

Use GitHub Issues for:

- bug reports;
- bounded feature requests;
- bounded questions about current repository behavior, contracts, or scope.

## Before opening an issue

Please check:

- `README.md`
- `docs/INDEX.md`
- `AGENTS.md`
- `CONTRIBUTING.md`
- `SECURITY.md` for security issues
- the relevant source, consumer, or implementation guide linked from
  `docs/INDEX.md`

## When not to use public issues

Do not use public issues for:

- vulnerability details;
- secrets, credentials, tokens, or private logs;
- private source data or private consumer state;
- anything that requires a private security handoff.

See `SECURITY.md` for the security reporting path.

## Response expectations

This is an early open source project with finite maintainer capacity. Clear
scope, exact file paths, reproducible steps, and validation output make issues
much easier to handle.
